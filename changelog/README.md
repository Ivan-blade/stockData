# 变更日志

## 文档结构

```
changelog/
├── README.md          ← 本文件，说明文档规范
├── 2026-06-28.md      ← 首日搭建，全量覆盖
└── ...
```

## 编写规范

1. **文件命名**：`YYYY-MM-DD.md`，按日期归档
2. **每个版本/重大更新**记录一次，包含：
   - 变更类型（feat/fix/perf/docs/refactor）
   - 关键数据变化（新增表、数据量变化）
   - 功能增减
   - 需要注意的兼容性问题
3. **语言**：中文
4. **示例**：

```markdown
## 2026-06-28

### feat: 全量数据覆盖
- A 股行情快照：5,516 只
- 港股行情快照：3,049 只
- 港股财务指标：2,644 家

### perf: 快照接口服务端分页
- 响应时间 0.8s → 0.06s
```
